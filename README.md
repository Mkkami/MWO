# Biletomat ZTM – Historie Użytkowników i Systemów

---

## 1. Historie dla użytkownika

1. **Szybki wybór biletu**  
   Jako użytkownik, chcę szybko wybrać rodzaj biletu, aby zminimalizować czas spędzony przy biletomacie.

2. **Wybór języka**  
   Jako użytkownik, chcę mieć możliwość wyboru języka, aby móc korzystać z biletomatu bez względu na znajomość języka lokalnego.

3. **Weryfikacja transakcji**  
   Jako użytkownik, chcę sprawdzić poprawność transakcji przed jej finalizacją, aby uniknąć pomyłek.

4. **Potwierdzenie zakupu**  
   Jako użytkownik, chcę otrzymać potwierdzenie zakupu (np. wydruk biletu lub elektroniczny bilet), aby móc korzystać z transportu zgodnie z przepisami.

5. **Wiele metod płatności**  
   Jako użytkownik, chcę płacić za bilet kartą, gotówką lub telefonem, aby mieć większą elastyczność w wyborze metody płatności.

6. **Instrukcje na ekranie**  
   Jako użytkownik, chcę otrzymać wyraźne instrukcje na ekranie, aby wiedzieć, jak dokonać zakupu krok po kroku.

7. **Licznik czasu**  
   Jako użytkownik, chcę widzieć czas pozostały na decyzję, aby móc szybko podjąć działanie.

---

## 2. Historie dla biletomatu (urządzenia)

1. **Automatyczna aktualizacja oferty**  
   Jako biletomat, chcę automatycznie aktualizować listę dostępnych biletów i ich cen, aby zapewnić zgodność z polityką przewoźnika.

2. **Rejestracja transakcji**  
   Jako biletomat, chcę rejestrować wszystkie transakcje i wysyłać raporty do systemu centralnego, aby umożliwić monitoring i kontrolę operacji.

3. **Czytelny ekran dotykowy**  
   Jako biletomat, chcę posiadać czytelny ekran dotykowy, aby użytkownik mógł łatwo nawigować po interfejsie.

4. **Obsługa różnych metod płatności**  
   Jako biletomat, chcę być wyposażony w terminal kart, czytnik gotówki i NFC, aby obsługiwać różnorodne transakcje.

5. **Wydawanie reszty**  
   Jako biletomat, chcę wydawać resztę w gotówce, jeśli użytkownik zapłaci nadmiarowo, aby transakcja była zgodna z oczekiwaniami.

---

## 3. Historie dla systemu transakcyjnego

1. **Status transakcji**  
   Jako system transakcyjny, chcę informować biletomat o statusie transakcji (sukces, odrzucenie, brak środków), aby użytkownik wiedział, co dalej robić.

2. **Dane księgowe**  
   Jako system transakcyjny, chcę rejestrować dane o transakcjach (czas, metoda płatności, kwota) dla celów księgowych i zgodności z przepisami.

3. **Autoryzacja kart**  
   Jako system transakcyjny, chcę autoryzować płatności kartą w czasie rzeczywistym, aby transakcje były szybkie i bezpieczne.

4. **Obsługa NFC i BLIK**  
   Jako system transakcyjny, chcę obsługiwać płatności NFC i BLIK, aby zapewnić nowoczesne metody płatności.

---

## 4. Historie dla systemu biletowego

1. **Aktualne taryfy**  
   Jako system biletowy, chcę dostarczać aktualne dane o taryfach i typach biletów do biletomatu, aby użytkownik miał zawsze poprawne informacje.

2. **Sprawdzanie ważności biletu**  
   Jako system biletowy, chcę umożliwiać sprawdzenie ważności biletu w czasie rzeczywistym, aby zapobiegać oszustwom.

3. **Rejestracja sprzedanych biletów**  
   Jako system biletowy, chcę rejestrować każde sprzedane bilety, aby śledzić ruch i sprzedaż w systemie.

4. **Integracja z aplikacją mobilną**  
   Jako system biletowy, chcę współpracować z aplikacjami mobilnymi, aby użytkownik mógł otrzymać bilet elektroniczny.

---

## 5. Historie dla systemu wsparcia technicznego

1. **Monitoring stanu biletomatów**  
   Jako system wsparcia technicznego, chcę automatycznie monitorować stan biletomatów (np. brak papieru, awaria terminala), aby reagować na problemy w krótkim czasie.

2. **Alerty serwisowe**  
   Jako system wsparcia technicznego, chcę wysyłać alerty o błędach i anomaliach do serwisu, aby uniknąć przestojów w działaniu urządzenia.

---

## 6. Historie dla administratora systemu

1. **Raporty sprzedaży**  
   Jako administrator, chcę mieć dostęp do raportów sprzedaży w czasie rzeczywistym, aby monitorować wyniki finansowe.

2. **Konfiguracja oferty**  
   Jako administrator, chcę konfigurować dostępne bilety, promocje i taryfy w systemie centralnym, aby odzwierciedlać zmiany w ofercie.

3. **Zdalne aktualizacje oprogramowania**  
   Jako administrator, chcę zdalnie aktualizować oprogramowanie biletomatów, aby zapewnić zgodność z nowymi wymogami i funkcjonalnościami.
